# Dpz.ServiceHub

一个用于 **本地开发环境服务管理与观测** 的 Avalonia 桌面工具。  
它帮助开发者快速启动、重建、停止和监控多个 ASP.NET Core 服务，包括 Redis 替代服务 Garnet、任务调度服务、认证中心、核心站点与 WebAPI。

---

## ✨ 项目简介

在开发环境中，通常需要同时启动多个服务（Web、API、任务调度、认证中心等），并且还要额外运行像 **Garnet** 这样的辅助服务。  
手动启动这些项目不仅繁琐，还难以实时查看输出日志或重建状态。

**Dpz.ServiceHub** 旨在解决这一痛点：  
它提供一个统一的桌面控制面板，让你可以一键启动、重建、停止并实时查看服务输出。

---

## 🚀 功能特性

| 功能 | 描述 |
|------|------|
| **服务管理** | 启动、停止、重启单个或全部服务 |
| **构建控制** | 一键重新构建解决方案或指定项目 |
| **实时日志输出** | 使用高性能文本控件（如 AvalonEdit）显示服务控制台输出 |
| **状态监控** | 实时显示服务运行状态（Running / Stopped） |
| **命令交互** | 支持向服务发送命令或脚本指令 |
| **配置管理** | 自定义服务路径、启动参数、环境变量 |
| **深色主题 UI** | 现代化界面设计，适合长时间开发使用 |

## 🧠 解决的痛点

- **启动繁琐**：多个项目需手动打开、编译、运行。  
  → 一键启动所有服务，自动检测端口与依赖。

- **日志分散**：各服务输出在独立控制台窗口中。  
  → 集中显示所有服务日志，支持筛选与颜色高亮。

- **重建不便**：修改代码后需手动重建多个项目。  
  → 支持一键重建指定项目或整个解决方案。

- **状态不可见**：无法直观看到哪些服务正在运行。  
  → 实时状态指示灯与进程监控。

## 🛠️ 技术栈

- **WPF (.NET 10)** — 桌面应用框架  
- **AvalonEdit** — 高性能文本显示控件  
- **Process API** — 启动与监控外部服务进程  
- **MVVM 架构** — 清晰分层，易于扩展  
- **Serilog** — 日志流式集成


## 📏 编码约定
### 命名规范
+ 私有字段成员使用 `_` 前缀，并使用驼峰命名
+ 参数、变量使用驼峰命名
+ 类、结构体、接口、方法、属性、事件等使用 Pascal 命名
+ 异步方法命名应该以 **Async** 结尾
+ 命名空间应该使用 `项目名.目录(.子目录)`
+ 命名空间使用文件作用域命名空间

### 类型与可空性
+ 按照语义严格遵循 Nullable（可空值类型、可空引用类型）
+ 返回单个对象时，需要根据语义返回可空引用类型或者不可空引用类型
+ 返回集合/数组时，除必要外（例如 `byte[]?`），如果没有数据都应该返回一个没有任何项的空数集合/数组
+ 参数类型应该尽可能的抽象，返回值的类型应该尽可能的具体
+ 语义冲突时，入参和出参应该分离，而不是共用一个类型
+ 不应该存在公开的字段

### 代码风格
+ 缩进使用 4 个空格
+ `if` `for` `foreach` `while` 等代码块，即使只有一行代码，也请使用大括号
+ 每行代码的最大长度：100 字符
+ 原则上，不得使用行尾注释
+ 如果只有一个构造函数的情况下，应该使用主构造函数

### 异步与性能
+ 异步方法的签名应该添加 `CancellationToken cancellationToken = default` 参数
+ `IEnumerable<T>` 类型不应该重复枚举

### 日志规范
+ 记录日志时，禁止拼接、内插字符串，而是应该使用结构化日志
